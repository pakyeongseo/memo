# 잉크 — 메모 (PWA)

서식을 그대로 보존하는 심플 메모장. 멀티 메모 + `.html` 저장/열기.

---

## 1. 깃허브 페이지에 올리기

기존 레포에 폴더 하나만 추가하면 됩니다. 경로는 전부 **상대경로**라 어느 서브폴더에 둬도 작동해요.

```
your-repo/
└── ink-memo-pwa/        ← 이 폴더를 통째로 넣기
    ├── index.html
    ├── manifest.json
    ├── service-worker.js
    ├── icon-192.png
    ├── icon-512.png
    └── favicon.png
```

```bash
git add ink-memo-pwa
git commit -m "add ink memo pwa"
git push
```

배포되면 접속 주소는:
`https://<아이디>.github.io/<레포명>/ink-memo-pwa/`

> 폴더 이름은 자유롭게 바꿔도 됩니다 (예: `memo`). 상대경로라 그대로 동작해요.

---

## 2. 안드로이드에 앱으로 설치

1. 크롬으로 위 주소 접속
2. 주소창 옆 **설치** 아이콘, 또는 메뉴(⋮) → **앱 설치 / 홈 화면에 추가**
3. 홈 화면에 잉크 아이콘이 생기고, 실행하면 주소창 없는 풀스크린 앱으로 뜸

설치하면 service worker가 동작해서 **오프라인에서도 열리고**, 데이터는 안정적인 HTTPS origin의 IndexedDB에 저장돼요.

---

## 3. `.html` 더블탭 → 이 앱으로 열기

PWA를 **설치한 뒤**, 파일 앱이나 다운로드 목록에서 `.html` 파일을 탭하면
"잉크"가 열기 옵션에 뜹니다. 선택하면 해당 파일이 새 메모로 불러와져요.

> 파일 핸들러(`file_handlers`)는 설치형 PWA에서만 작동합니다. 브라우저 탭으로 열어둔 상태에서는 앱 안의 **열기** 버튼을 쓰세요.

---

## 4. 업데이트 배포할 때

`index.html` 등을 수정해서 다시 push할 때는,
`service-worker.js` 맨 위의 캐시 버전만 올려주세요.

```js
const CACHE = "ink-memo-v1";   // → "ink-memo-v2"
```

그래야 기존 캐시가 정리되고 새 파일로 갱신됩니다. (안 올리면 옛 버전이 계속 캐시될 수 있어요.)

---

## 기능 요약

- 여러 메모 생성 / 이름변경 / 복제 / 삭제 / 검색
- 붙여넣기 시 **원본 서식 그대로 보존** (볼드·색상·링크·이미지 등)
- 입력 즉시 자동저장 (IndexedDB)
- `.html`로 저장 (완결형 HTML 문서)
- `.html` 열기 (불러온 메모도 서식 유지, `<script>`는 자동 제거)
- 라이트 / 다크 테마
